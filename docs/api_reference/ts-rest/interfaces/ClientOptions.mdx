title: "Client Options"
description: "Configuration settings for the MemMachineClient."
---

## Interface `ClientOptions`

| Property | Type | Description |
| :--- | :--- | :--- |
| `apiKey` | `string` | **Required.** Your MemMachine API key for authentication. |
| `baseUrl` | `string` | *Optional.* The URL of your MemMachine instance. Defaults to production. |
| `timeout` | `number` | *Optional.* Request timeout in milliseconds. |

---

## Interface `HealthStatus`

Returned by `client.healthCheck()`.

| Property | Type | Description |
| :--- | :--- | :--- |
| `status` | `string` | The current health state (e.g., "healthy"). |
| `version` | `string` | The version of the MemMachine server. |
| `uptime` | `number` | Server uptime in seconds. |

---

## Usage Example

Use `ClientOptions` to connect, then use `HealthStatus` to verify your environment is ready.

```typescript
import { MemMachineClient } from '@memmachine/sdk';

const client = new MemMachineClient({ 
  base_url: 'your-base-url',
  api_key: 'your_secret_key'
});

async function checkConnection() {
  try {
    const health = await client.healthCheck();
    
    if (health.status === 'healthy') {
      console.log(`Connected to ${health.service} v${health.version}`);
      console.log(`Episodic Manager: ${health.memory_managers.episodic_memory ? 'UP' : 'DOWN'}`);
    }
  } catch (error) {
    console.error("Connection failed:", error.message);
  }
}

checkConnection();
```
