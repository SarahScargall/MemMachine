## Interface `AddMemoryOptions`

The `AddMemoryOptions` interface defines the payload for the `client.addEpisodicMemory()` method. It requires both the content of the memory and the metadata context for proper isolation.

| Property | Type | Description |
| :--- | :--- | :--- |
| `content` | `string` | **Required.** The raw text content of the interaction or fact to be stored. |
| `metadata` | [`ProjectContext`](./ProjectContext) | **Required.** Scoping identifiers including `orgId` and `projectId`. |
| `role` | `MemoryProducerRole` | *Optional.* The role of the entity that produced the memory (e.g., `user`, `assistant`, or `system`). |


## Usage Example

The following example demonstrates how to structure the options when adding a memory directly through the `MemMachineClient`.

```typescript
import { MemMachineClient, AddMemoryOptions } from '@memmachine/sdk';

const client = new MemMachineClient({ apiKey: process.env.MEMMACHINE_API_KEY });

const memoryPayload: AddMemoryOptions = {
  content: "The user expressed a strong preference for TypeScript over vanilla JavaScript.",
  metadata: {
    orgId: "dev_studio_01",
    projectId: "coding_assistant_v2",
    userId: "dev_user_77"
  },
  role: "user"
};

await client.addEpisodicMemory(memoryPayload);
```
<Note> Ensure the metadata object contains a valid projectId. Memories added without a project context will fail validation to prevent data leakage between workspaces. </Note>
