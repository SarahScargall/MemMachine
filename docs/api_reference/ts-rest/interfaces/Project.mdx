## Interface `Project`

The `Project` object represents the high-level container for your data. It defines the identity and the specific AI models (embedder/reranker) used for semantic vectorization and retrieval within a workspace.

| Property | Type | Description |
| :--- | :--- | :--- |
| `id` | `string` | **Required.** The unique identifier for the project. |
| `orgId` | `string` | **Required.** The organization ID this project belongs to. |
| `description` | `string` | *Optional.* A brief summary of the project's purpose. |
| `config` | `ProjectConfig` | *Optional.* Technical settings for the project's AI engine. |

### `ProjectConfig`

| Property | Type | Description |
| :--- | :--- | :--- |
| `embedder` | `string` | The model used for generating semantic embeddings. |
| `reranker` | `string` | The model used for optimizing search results. |

## Usage Example

In the current SDK, project data is typically retrieved through the primary client. The following example demonstrates how to list and inspect projects using the updated naming conventions.

```typescript
import { MemMachineClient, Project } from '@memmachine/sdk';

const client = new MemMachineClient({ 
  apiKey: process.env.MEMMACHINE_API_KEY 
});

// Retrieve all projects associated with your API key
const projects: Project[] = await client.getProjects();

projects.forEach((project) => {
  console.log(`Project ID: ${project.id}`);
  console.log(`Embedder: ${project.config?.embedder ?? 'Default'}`);
});
```

<Note title="Property Refactor"> The properties project_id and org_id have been updated to id and orgId to align with the standard TypeScript camelCase convention used throughout the current SDK. </Note>
