## Enumerations

These enums provide type safety for memory classification and entity roles within the MemMachine ecosystem.

### `MemoryType`

Defines the storage and retrieval logic applied to a specific memory entry.

| Member | Value | Description |
| :--- | :--- | :--- |
| `episodic` | `"episodic"` | Short-term, chronological conversation logs. |
| `profile` | `"profile"` | Long-term, synthesized user facts and preferences. |

### `MemoryProducerRole`

Identifies the entity responsible for generating the memory content.

| Member | Value | Description |
| :--- | :--- | :--- |
| `user` | `"user"` | Content provided by the human end-user. |
| `assistant` | `"assistant"` | Content generated by the AI agent. |
| `system` | `"system"` | Internal instructions or environmental context. |

## Constants

### `DefaultConfig`

Standard configuration values applied when optional parameters are omitted from the `MemMachineClient`.

| Constant | Value | Description |
| :--- | :--- | :--- |
| `defaultBaseUrl` | `"https://api.memmachine.com"` | The production API endpoint. |
| `defaultTimeout` | `30000` | Default request timeout in milliseconds (30s). |

---

## Usage Example

Using enums ensures that your application remains resilient to string-based errors.

```typescript
import { MemMachineClient, MemoryProducerRole } from '@memmachine/sdk';

const client = new MemMachineClient({ apiKey: '...' });

await client.addEpisodicMemory({
  content: "User requested a summary of last week's notes.",
  metadata: { 
    orgId: "org_1", 
    projectId: "proj_A" 
  },
  role: MemoryProducerRole.user // Using the enum for type safety
});
```

<Note title="Type Safety"> Always prefer using the exported Enums over raw strings. This allows the TypeScript compiler to catch typos during development rather than at runtime. </Note>
