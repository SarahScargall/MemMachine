## Class `MemMachineClient`

The `MemMachineClient` is the main API client for interacting with the MemMachine service. Following the latest SDK updates, this client now serves as the direct interface for episodic and profile memory operations using **Metadata Scoping**

```typescript
new MemMachineClient(options: ClientOptions)
```

**Parameters**

| **Name**  | **Type**                                                     | **Description**                                 |
| --------- | ------------------------------------------------------------ | ----------------------------------------------- |
| `options` | [`ClientOptions`](../interfaces/ClientOptions) | Configuration including `apiKey` and `baseUrl`. |

------

## Active Memory Methods

These methods are the primary way to interact with the memory engine. Every call requires a `metadata` object to maintain organizational and project context.

### `addEpisodicMemory()`

Adds a new chronological memory to the system.

```typescript
await client.addEpisodicMemory({
  content: string,
  metadata: ProjectContext
})
```

**Parameters**

- `content`: The raw text of the memory to store.
- `metadata`: An object containing `orgId`, `projectId`, and optionally `userId`.

  

  

**Throws**

- [`MemMachineAPIError`](./MemMachineAPIError) if the ingestion fails.

  

  

------

### `searchProfileMemory()`

Performs a semantic search to retrieve synthesized facts and user profile insights.

```typescript
await client.searchProfileMemory({
  query: string,
  metadata: ProjectContext,
  limit?: number
})
```

**Parameters**

- `query`: The natural language question or search term.
- `metadata`: Contextual identifiers for scoping the search.
- `limit`: (Optional) Maximum number of results to return.

## Utility Methods

### `getProjects()`

Retrieves a list of all projects accessible to the current API key.

```typescript
await client.getProjects()
```

### `healthCheck()`

Checks the connection status and uptime of the MemMachine server.

```typescript
const status = await client.healthCheck();
```

## Usage Example

```typescript
import { MemMachineClient } from '@memmachine/sdk';

const client = new MemMachineClient({ 
  apiKey: process.env.MEMMACHINE_API_KEY 
});

async function main() {
  // Direct interaction pattern
  await client.addEpisodicMemory({
    content: "User prefers dark mode and high-contrast settings.",
    metadata: { 
      orgId: "my_org", 
      projectId: "ui_prefs_project",
      userId: "user_123" 
    }
  });

  const profile = await client.searchProfileMemory({
    query: "What are the user's UI preferences?",
    metadata: { projectId: "ui_prefs_project" }
  });
}
```

<Note> The `.project()` method still exists for legacy support but is no longer the recommended path. For all new implementations, use the direct memory methods shown above. </Note>
