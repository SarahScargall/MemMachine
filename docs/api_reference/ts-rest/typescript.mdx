The MemMachine TypeScript SDK is built for modern AI workflows, providing a type-safe, metadata-driven interface to the MemMachine memory engine. 

## Active Entry Point

The latest version of the SDK leverages a unified client structure. Instead of managing multiple class instances, you perform all memory operations through the `MemMachineClient` using **Metadata Scoping**.

## Quick Start

<Steps>
  <Step title="Install">
    ```bash
    npm install @memmachine/sdk
    ```
  </Step>

  <Step title="Initialize & Use">
    Supply your `orgId` and `projectId` within the `metadata` object of your calls to maintain context.

    ```typescript
    import { MemMachineClient } from '@memmachine/sdk';

    const client = new MemMachineClient({ apiKey: 'your_api_key' });

    // Active pattern: Direct client calls with metadata
    await client.addEpisodicMemory({
      content: "User likes high-contrast UI.",
      metadata: { orgId: "org_1", projectId: "proj_A", userId: "user_1" }
    });
    ```
  </Step>
</Steps>

## Contributing & Development

If you are looking to contribute to the SDK or run local tests, follow these steps:

1. **Install Dependencies:** `npm install`
2. **Build the Package:** `npm run build`
3. **Run Unit Tests:** `npm run test`
4. **Try the Demo:** Execute `npm run example` to see a local script in action.
